# CodeCraftHub2
CodeCraftHub - Courses API
CodeCraftHub is a Spring Boot REST API that provides CRUD operations for courses. Data is persisted to a JSON file (courses.json) in the project directory. It uses Jackson for JSON processing, auto-generates IDs, and validates required fields and status values.

Features

Create, Read, Update, Delete (CRUD) operations for courses
Data stored in a JSON file named courses.json
Auto-generated id (starting from 1) and createdAt timestamp
Target date validated in YYYY-MM-DD format
Status values strictly enforced: Not Started, In Progress, Completed
Automatic creation of courses.json if it does not exist
Basic error handling for missing fields, not found, invalid status, and file I/O errors
JSON processing with Jackson (Java Time support for date fields)
Tech stack

Spring Boot
Jackson (JSR-310 for Java 8+ date/time)
Maven for dependency management
Project structure

Course.java: Model class
CourseService.java: Service layer handling business logic and file I/O
CourseController.java: REST controller with endpoints
CodeCraftHubApplication.java: Main Spring Boot application
pom.xml: Maven dependencies
Optional: Custom exception classes for clean error handling
Prerequisites

JDK 17 or newer
Maven 3.x
(Optional) IDE like IntelliJ IDEA, VS Code, etc.
Installation

Clone the repository
git clone https://your-repo-url/code-crafthub.git
Navigate to the project directory
cd code-crafthub
Build the project (compiles the code and packages the app)
mvn clean package
Run the application
Run via Maven: mvn spring-boot:run
Or run the packaged JAR: java -jar target/<your-artifact-id>.jar
The default port is 8080
How to run

By default, the app starts on http://localhost:8080
It will automatically create courses.json in the project root if it doesn’t exist
API Endpoints Base URL: http://localhost:8080/api/courses

POST /api/courses Create a new course Request body (JSON): { "name": "Intro to Java", "description": "Learn the basics of Java", "targetDate": "2026-12-31", "status": "Not Started" } Response:

201 Created
Body: the created course with generated id and createdAt
GET /api/courses Get all courses Response:

200 OK
Body: Array of Course objects
GET /api/courses/{id} Get a specific course by ID Response:

200 OK
Body: Course object
404 Not Found if not exist
PUT /api/courses/{id} Update a course Request body (JSON): { "name": "Intro to Java - Updated", "description": "Updated description", "targetDate": "2027-01-15", "status": "In Progress" } Response:

200 OK
Body: updated Course object
400 Bad Request for validation errors
404 Not Found if course not found
DELETE /api/courses/{id} Delete a course Response:

204 No Content on success
404 Not Found if not found
Request/Response details

Required fields for create/update:
name: string (non-empty)
description: string (non-empty)
targetDate: date (YYYY-MM-DD)
status: one of "Not Started", "In Progress", "Completed"
id is auto-generated on create (not provided by client)
createdAt is auto-generated on create
Status validation strictly enforces allowed values to prevent invalid states
Error handling (how errors are returned)

400 Bad Request: Missing or invalid fields (e.g., missing name/description, invalid date format, invalid status)
404 Not Found: When a course with the specified ID does not exist
500 Internal Server Error: File I/O or storage-related problems (e.g., cannot read/write courses.json)

Storage behavior

The app uses a file named courses.json in the project root.
If the file does not exist, it is created automatically and initialized with an empty array [].
IDs are generated by taking the max existing id and adding 1.
targetDate is stored as a date in format YYYY-MM-DD.
createdAt is stored as an ISO timestamp (e.g., 2024-08-12T14:23:45).
Troubleshooting

Server fails to start or port is already in use
Ensure no other process is occupying port 8080. Change application port in application.properties if needed.
400 Bad Request on create/update
Check required fields are present and non-empty.
Ensure targetDate is in YYYY-MM-DD format (e.g., 2026-12-31).
Ensure status is one of: Not Started, In Progress, Completed.
404 Not Found for GET/PUT/DELETE
Verify the ID exists in courses.json or create a new one.
File I/O errors (reading/writing courses.json)
Check file permissions for the project directory.
Ensure the app has permission to create and modify courses.json in the project root.
JSON parsing issues
Ensure you’re sending valid JSON with correct field names and types.
Data inconsistency
If you manually edit courses.json, ensure it remains a valid JSON array of Course objects.
Tips

You can inspect the course records by calling GET /api/courses to see the current state after operations.
If you want to customize the storage location, modify the STORAGE_FILE path in CourseService.java (not recommended for production without proper configuration).
